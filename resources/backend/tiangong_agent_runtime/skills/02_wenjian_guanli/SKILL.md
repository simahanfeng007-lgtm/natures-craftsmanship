# 文件管理

## 1. 定位
文件管理是临渊者执行链的入口级Skill。负责建立路径事实、识别项目结构、读取目标文件、搜索证据片段、计算文件哈希、受控写入，为文档处理、代码诊断、终端执行、打包、收口质检提供可靠文件上下文。

## 2. 触发条件
- 用户要求查看目录、检查包体、找文件、读文件
- 用户上传 zip / txt / py / md / json / yaml / xlsx / docx 等文件后要求分析
- 代码诊断前需要建立文件树
- 打包前需要生成文件清单或 hash
- 文档处理前需要定位源文件
- 用户要求"看看这个文件 / 里面有什么 / 这个包有什么"
- 路径不确定，需要先探索目录结构

## 3. 反触发条件
- PDF/Word/Excel/PPT 的结构化解析 → 文档处理
- Python 测试、编译、命令执行 → 终端执行
- 代码 bug 根因分析 → 代码诊断（文件管理可负责取证）
- zip 发布包生成 → 项目打包
- 项目最终验收 → 项目收口质检
- 没有文件操作需求，纯聊天 → 不触发

## 4. 输入契约
- workspace/project_root：工作区或项目根路径
- 目标路径或文件名
- 搜索关键词（如需要）
- 是否允许写入：是/否
- 写入目标路径（如允许写入）
- 是否需要 hash：是/否
- 是否需要递归扫描：是/否

路径规则：
- 优先使用用户上传文件路径或工作区路径
- 不确定路径时先 list_dir / scan_project，禁止凭空编造路径
- Windows 反斜杠和 POSIX 正斜杠都需规范化处理
- 相对路径必须说明解析基准（通常为 workspace）

## 5. 输出契约
每次任务至少输出：
- 已访问路径列表
- 文件存在性结论
- 读取/搜索到的关键证据（含行号或片段位置）
- 未能读取的原因（如权限、编码、不存在）
- 下一步建议（转哪个Skill或继续什么操作）

涉及写入时额外输出：
- 写入路径、写入类型（新建/覆盖/追加）
- 变更摘要
- 是否需要用户确认（覆盖已有文件时强制确认）
- hash 或可回滚信息

## 6. 工具选择矩阵

**list_dir**
- 用途：查看指定目录直接子项
- 适用：路径不确定、需确认目录结构
- 失败恢复：路径不存在 → 回退上级目录或要求重新定位

**scan_project**
- 用途：生成项目级文件树
- 适用：zip 解包、项目质检、代码诊断前
- 失败恢复：缩小扫描范围，排除 node_modules、venv、dist、build 等大目录

**read_file**
- 用途：读取明确目标文件
- 适用：用户指定文件、搜索命中文件、核心配置文件
- 失败恢复：尝试编码降级（UTF-8→GBK→Latin-1）、片段读取（limit/offset）、确认路径大小写

**workspace_text_search**
- 用途：按关键词搜索工作区文本
- 适用：定位函数、配置项、错误信息、版本号
- 失败恢复：改用更短关键词、同义关键词、加文件后缀限制

**file_sha256**
- 用途：生成文件完整性证据
- 适用：打包前、修改前后对比、交付验收
- 失败恢复：确认文件存在和权限

**write_workspace_file**
- 用途：受控写入工作区文件
- 适用：用户明确要求生成 txt、报告、补丁文件、配置文件
- 硬约束：写入前必须明确目标路径和目的；覆盖已有文件须说明覆盖风险并请求确认；禁止越权写入系统路径
- 失败恢复：改写到可用输出目录；保留失败原因

**make_dir**
- 用途：在当前工作区内创建文件夹
- 适用：整理桌面、建立分类目录、准备移动目标目录
- 约束：需要 workspace_full 权限；不得越过工作区

**move_path**
- 用途：在当前工作区内移动或重命名文件/文件夹
- 适用：整理桌面、合并目录、归类文件、重命名错误文件
- 参数：source、target、overwrite
- 约束：需要 workspace_full 权限；移动后必须 list_dir 验证

**copy_path**
- 用途：在当前工作区内复制文件/文件夹
- 适用：备份、保留副本后再整理
- 参数：source、target、overwrite
- 约束：需要 workspace_full 权限；复制后必须 list_dir 或 file_sha256 验证

**delete_path**
- 用途：删除当前工作区内文件/文件夹
- 适用：用户明确要求清理、整理、删除垃圾文件或缓存目录
- 参数：path、recursive
- 约束：需要 workspace_full 权限；不得删除工作区根目录；删除目录前确认目标是应清理目录

## 7. 标准执行流程

**阶段 A：路径预检**
- 解析用户意图，明确根目录
- 路径不确定 → list_dir 或 scan_project
- 路径确定 → 进入阶段 B

**阶段 B：证据获取**
- 读明确文件 → read_file
- 不知道文件位置 → workspace_text_search
- 需要结构清单 → scan_project

**阶段 C：证据压缩**
- 大文件只提取相关片段（用 limit/offset）
- 输出路径、行号或片段位置
- 不用无关全文污染上下文

**阶段 D：受控写入**
- 仅用户明确要求生成/修改文件时触发
- 写入前确认路径、内容类型和覆盖风险
- 覆盖已有文件必须请求用户确认
- 写入后 file_sha256 或再次读取验证

**阶段 E：收口**
- 输出完成情况、证据、失败点、下一步建议

## 8. 证据链规则
- "文件存在/不存在"结论必须基于工具返回，不得猜测
- "代码里有这个问题"必须给文件路径和内容片段
- "已经写入"必须给路径和写入摘要
- 打包前后用 hash 辅助确认完整性
- 不允许只凭文件名推断内容

## 9. 失败恢复
- path_not_found：回退上级目录、扫描根目录、提示可能路径
- encoding_error：尝试其他编码（GBK、Latin-1）或标记为二进制
- permission_denied：不绕过权限，输出受限路径和建议
- file_too_large：改用 workspace_text_search、片段读取
- empty_result：换关键词、扩大搜索范围、list_dir 确认目录
- write_failed：改到可写目录，输出失败原因

## 10. 质量门
- 通过：找到目标文件并给出证据 / 成功写入并验证
- 部分通过：找到部分文件或证据不足但说明了限制
- 不通过：路径不明、工具失败且无降级证据
