# 文档处理

## 1. 定位
文档处理负责所有非纯代码文件的结构化理解、内容抽取、问答、改写、导出和回滚。面向 PDF、Word、Excel、PPT、Markdown、纯文本等文档。不是简单"读文档"，而是尽量保留页码、标题层级、表格结构、批注、公式、目录等结构化信息。

## 2. 触发条件
- 用户上传文档后要求总结、分析、抽取表格、找问题
- 用户要求根据文档生成报告、清单、会议纪要、课程稿、合同审阅
- 用户要求改写文档、调整结构、导出格式
- 用户问"这个 PDF 里说了什么""这个 Excel 怎么样""这份 Word 有没有问题"
- 用户要求抽取表格数据、对比文档、校对内容

## 3. 反触发条件
- 代码仓库诊断 → 代码诊断
- 目录/文件存在性检查 → 文件管理
- 命令执行、转换脚本运行 → 终端执行
- 最终发布包质量验收 → 项目收口质检
- 纯文本聊天中提到的文档（未上传）→ 先要求上传

## 4. 输入契约
- 文档路径
- 文档类型（PDF/Word/Excel/PPT/Markdown/TXT）
- 用户目标：总结 / 问答 / 抽取 / 改写 / 导出 / 校对 / 回滚
- 是否允许修改原文档
- 目标输出格式：txt / md / docx / xlsx / pptx / pdf
- 需要保留的结构：页码、标题、表格、批注、公式、图片说明

## 5. 输出契约

**总结类**
- 文档范围（多少页/sheet/段落）
- 主题摘要
- 关键结论
- 证据页码/段落/表格位置
- 不确定项清单

**问答类**
- 答案
- 引用页码/段落/表格位置
- 未找到时明确说明"未找到"

**抽取类**
- 字段表
- 来源位置（页码/sheet名/行列）
- 置信度/异常项标记

**改写类**
- rewrite_plan（改写计划）
- 变更摘要
- 修改范围
- 可回滚点

**导出类**
- 输出路径、格式
- 是否保留结构
- 导出校验结果

## 6. 工具选择矩阵

**document_parse**
- 用途：完整解析文档结构（目录、标题、段落、表格）
- 适用：首次打开文档、需要整体结构
- 失败恢复：转 document_text_extract 或提示扫描件/加密/损坏

**document_query**
- 用途：按问题或关键词检索相关内容
- 适用：用户问具体问题、查条款、查数据
- 失败恢复：换关键词、扩大页码范围、改用 text_extract

**document_text_extract**
- 用途：抽取纯文本
- 适用：结构解析失败、只需文本摘要
- 失败恢复：标记结构丢失，不得伪称保留表格

**document_rewrite_plan**
- 用途：生成改写计划
- 适用：任何实际改写之前（不可跳过）
- 约束：没有 rewrite_plan 时不得执行改写

**document_apply_rewrite**
- 用途：执行受控改写
- 适用：用户明确要求修改且计划已确认
- 失败恢复：保留失败位置，回滚或输出人工修改建议

**document_export**
- 用途：导出目标格式
- 适用：报告、清单、整理稿生成
- 失败恢复：降级到 md/txt 或说明格式限制

**document_rollback**
- 用途：恢复改写前状态
- 适用：改写失败、用户撤销、验收不通过

## 7. 标准执行流程

**阶段 A：文档识别**
- 确认文件类型、路径、用户目标
- 不清楚路径 → 转 02_wenjian_guanli

**阶段 B：结构解析**
- 优先 document_parse
- 只需问答 → document_query
- parse 失败 → 降级 document_text_extract，标记结构丢失

**阶段 C：证据抽取**
- 答案必须绑定页码、段落、表格位置或 sheet 名称
- 表格抽取保留表头、行列、单位、异常空值
- 不确定内容标记"不确定/解析不足/未找到"

**阶段 D：生成/改写**
- 总结与问答：先证据后回答
- 改写：先 rewrite_plan → 再 apply_rewrite
- 导出：先确认目标格式 → export

**阶段 E：校验与回滚**
- 改写后必须说明改动位置
- 失败 → document_rollback 或提示可回滚点

## 8. 文档类型专项规则

**PDF**
- 区分文本PDF、扫描PDF、论文PDF、合同PDF
- 必须带页码证据
- 扫描件OCR不可靠时输出低置信度
- 解析失败不得虚构内容

**Word**
- 注意标题层级、批注、修订痕迹
- 改写前必须生成 rewrite_plan

**Excel**
- 保留 sheet 名、表头、行列位置
- 区分公式单元格和数值单元格
- 不能把多个 sheet 混成一个表

**PPT**
- 按页序处理
- 区分标题、正文、备注、图表

**Markdown/TXT**
- 保留标题层级、列表、代码块

## 9. 证据链规则
- 不允许"文档里说了"但不给页码/段落/表格位置
- 不确定内容必须标记"不确定 / 解析不足 / 未找到"
- 结构丢失时必须说明降级方式
- 表格抽取必须保留行列定位

## 10. 失败恢复
- 文件不存在：转 02_wenjian_guanli 定位
- 文档加密：说明不能解析
- 扫描件 OCR 不可靠：输出低置信度
- 表格错位：保留原始片段并标记疑似错位
- 公式丢失：提示公式解析不完整
- 导出失败：降级为 md/txt
- 改写失败：document_rollback

## 11. 质量门
- 通过：回答/摘要/抽取均有证据；改写有计划+执行+校验
- 部分通过：解析部分成功，但结构/表格/公式存在限制
- 不通过：无法读取文档或证据不足却生成确定结论

## 13. 验收用例
1. PDF 总结：要求带页码证据
2. Word 合同审阅：要求风险点和条款位置
3. Excel 多 sheet 抽取：要求 sheet 名和行列
4. PPT 课程稿提炼：要求按页输出
5. 扫描 PDF 解析失败：要求降级说明
6. 文档改写后回滚
7. 导出 txt/md/docx 失败时降级
