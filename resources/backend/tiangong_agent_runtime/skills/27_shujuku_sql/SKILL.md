# 数据库与SQL

面向sqlite、业务数据库、只读查询、schema检查、SQL解释、导入导出规划场景的技能。目标：让 LLM 安全理解数据库结构并优先执行只读查询。

## 什么时候用
- 用户给出数据库文件、SQL、表结构、查询需求时使用
- 需要查询客户池、知识库、业务表、CRM数据时使用

## 反触发条件（可选）
- 普通 Excel/CSV 分析转交数据表格分析
- API 数据查询转交 API 接口调用
- 写库/迁移必须先做计划和确认

## 标准流程
1. 明确用户目标、输入材料、风险等级和最终交付物。
2. 判断是否已有足够证据；证据不足时先读取、解析、检索或生成计划。
3. 优先调用只读工具形成证据，再调用写入/执行类工具。
4. 对结果做一致性检查：字段、来源、时间、路径、数量和可复现性。
5. 输出中文结论，区分已验证、推断、风险和未验证项。
6. 如需写入、提交、批量执行或影响外部系统，必须进入确认或只生成计划。
7. 交付时给出结果、证据、产物路径、下一步建议和回滚/复核方式。
