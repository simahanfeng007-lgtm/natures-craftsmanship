# 增长实验与复盘

面向A/B测试、增长实验、uplift、bandit、复盘、活动结果分析和预算再分配场景的技能。目标：用实验和因果增量思路优化运营动作，避免只凭感觉做增长。

## 什么时候用
- 用户要求做A/B测试、增长实验、复盘、投放优化、预算分配、验证哪个方案更有效时使用
- 需要判断一个运营动作是否真的带来增量时使用

## 反触发条件（可选）
- 样本过小时不能下强结论
- 真实投放和预算调整前必须确认
- 归因数据不足时转交RevOps数据与归因

## 标准流程
1. 明确业务目标、目标客户、当前阶段、输入数据、风险等级和最终交付物。
2. 先识别客户旅程位置：获客、激活、培育、诊断、方案、成交、复盘。
3. 优先调用只读分析工具形成证据，再生成计划、话术、评分或复盘。
4. 对线索、机会、渠道、内容、触点和成交风险分别输出原因与置信度。
5. 涉及真实触达、批量发送、CRM写入、报价或预算调整时，只生成计划并要求确认。
6. 交付时给出结论、证据、下一步动作、责任人建议、时间节奏和未验证项。
