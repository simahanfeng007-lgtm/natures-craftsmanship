# RevOps数据与归因

面向CRM、pipeline、渠道归因、销售漏斗、机会胜率、收入看板和运营指标场景的技能。目标：统一销售、市场、客户成功数据，判断收入链路真实瓶颈和渠道贡献。

## 什么时候用
- 用户要求分析CRM、漏斗、pipeline、渠道贡献、成交率、销售预测、看板时使用
- 需要判断哪个渠道或动作带来转化时使用

## 反触发条件（可选）
- 没有数据时只能生成看板/归因计划
- 单个客户跟进转交销售转化
- 批量数据清洗结合数据表格分析

## 标准流程
1. 明确业务目标、目标客户、当前阶段、输入数据、风险等级和最终交付物。
2. 先识别客户旅程位置：获客、激活、培育、诊断、方案、成交、复盘。
3. 优先调用只读分析工具形成证据，再生成计划、话术、评分或复盘。
4. 对线索、机会、渠道、内容、触点和成交风险分别输出原因与置信度。
5. 涉及真实触达、批量发送、CRM写入、报价或预算调整时，只生成计划并要求确认。
6. 交付时给出结论、证据、下一步动作、责任人建议、时间节奏和未验证项。
