# 网络小说完整生产流水线

面向网络小说从开书、设定、章节生产、文风成稿、连载复盘到终审核改的统一生产场景。目标：把原本分散的类型、爽点、人物、世界观、文风、留存和终审能力压成一条可连续执行的小说生产线。

## 什么时候用
- 用户要求创作、续写、改写、质检、规划网络小说时使用。
- 用户说“写一章”“做开书设定”“生成大纲”“优化文风”“检查爽点”“去AI腔”“做连载计划”时使用。
- 用户需要把故事工程、文风声纹、章节成稿、修订质检合成一个结果时使用。
- 默认优先调用 `wangwen_novel_factory_run`，不要拆开调用旧的 35-42 或 53 文风碎片能力。

## 反触发条件（可选）
- 纯文件读写、打包、代码修复、联网搜索、办公排版，应转交对应基础技能。
- 用户只要求普通公文、营销文案、论文写作，不属于网络小说生产线。
- 用户要求模仿特定在世作者文风时，不做仿写，只抽象为类型文风和句法特征。
- 涉及真实发布、投稿、签约、付费平台提交时，只生成计划或草稿，不直接执行外部动作。

## 标准流程
1. 调用 `wangwen_novel_factory_run` 建立完整生产线，输出类型承诺、读者期待、故事圣经、章节生产简报、文风卡、人物声线和终审规则。
2. 如果只做开书设定，使用 `wangwen_novel_bible_build` 补强故事圣经。
3. 如果只做单章，使用 `wangwen_chapter_brief_build` 输出本章成稿简报，并由 LLM 按简报直接写正文。
4. 成稿后使用 `wangwen_draft_quality_check` 诊断节奏、文风、AI腔、套话、人物声线和留存风险。
5. 对未通过内容使用 `wangwen_revision_plan_build` 生成修订补丁，再由 LLM 输出最终稿。
6. 输出结论、正文/修订稿、质量风险、下一章推进建议和故事圣经更新点。

## 工具
- `wangwen_novel_factory_run`
- `wangwen_novel_bible_build`
- `wangwen_chapter_brief_build`
- `wangwen_draft_quality_check`
- `wangwen_revision_plan_build`
