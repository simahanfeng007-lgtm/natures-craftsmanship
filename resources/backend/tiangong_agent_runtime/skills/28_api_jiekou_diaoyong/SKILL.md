# API接口调用

面向HTTP API、OpenAPI、Webhook、鉴权检查、响应字段抽取场景的技能。目标：把外部系统接口调用变成可审计、可验证、可降级的工具规格。

## 什么时候用
- 用户要求对接飞书、企微、CRM、ERP、知识库、Webhook、第三方接口时使用
- 用户给出 API 文档、URL、JSON 响应、错误码时使用

## 反触发条件（可选）
- 普通网页浏览转交浏览器自动化
- 数据库查询转交数据库与SQL
- 涉及密钥时只检查结构，不输出密钥

## 标准流程
1. 明确用户目标、输入材料、风险等级和最终交付物。
2. 判断是否已有足够证据；证据不足时先读取、解析、检索或生成计划。
3. 优先调用只读工具形成证据，再调用写入/执行类工具。
4. 对结果做一致性检查：字段、来源、时间、路径、数量和可复现性。
5. 输出中文结论，区分已验证、推断、风险和未验证项。
6. 如需写入、提交、批量执行或影响外部系统，必须进入确认或只生成计划。
7. 交付时给出结果、证据、产物路径、下一步建议和回滚/复核方式。
